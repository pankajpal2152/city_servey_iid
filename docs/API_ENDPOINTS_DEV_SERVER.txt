CITY SURVEY DEV SERVER API ENDPOINTS
Date: 2026-07-09

Swagger URL:
https://api-dev.citysurveyors.com.sg/api/api-docs/

Base API URL for frontend:
https://api-dev.citysurveyors.com.sg

Suggested frontend env value:
VITE_API_BASE_URL=https://api-dev.citysurveyors.com.sg

Common JSON headers:
Content-Type: application/json
Accept: application/json

Protected API header:
Authorization: Bearer <Token>

Get token from:
POST https://api-dev.citysurveyors.com.sg/api/auth/api-post-authenticate-user

ACTIVE API URL LIST

1. Health Check
Method: GET
URL: https://api-dev.citysurveyors.com.sg/health
Auth Required: No

2. Authenticate User
Method: POST
URL: https://api-dev.citysurveyors.com.sg/api/auth/api-post-authenticate-user
Auth Required: No
Body:
[
  {
    "USER_NAME": "sai@yopmail.com",
    "PASSWORD": "Abc@1234"
  }
]

NOTES

- Use the dev base URL above in frontend development builds.
- Do not hardcode DB credentials, JWT secret, or static tokens in frontend code.
- Login response returns the JWT in the field named Token.
- Send Authorization: Bearer <Token> only after successful login.
- The full project payload is a write operation and may create dev DB records.
- The full master system role payload is a write operation and may create dev
  DB records.
- The full master survey type payload is a write operation and may create dev
  DB records.
- The full master property type payload is a write operation and may create dev
  DB records.
- Full request/response examples are available in docs/API_URLS.md.
