## Important Technical Finding

The database developer's sample for `USP_POST_USER_PROFILE_ACTIVITY` showed a
plain JSON object, but the live stored procedure reads fields using `$[0]`.

Backend decision:

- Frontend can send a normal object.
- Backend wraps the object as a one-item array before calling the stored
  procedure.

Implemented in:

- `routes/user.js`

## Current Working API Contracts

### Authenticate User

Endpoint:

`POST /api/auth/api-post-authenticate-user`

Request:

```json
[
  {
    "USER_NAME": "sai@yopmail.com",
    "PASSWORD": "Abc@1234"
  }
]
```

Response:

- Returns DB user data
- Returns `Token` when `status` is `true`

### Add or Update User

Endpoint:

`POST /api/user/api-post-add-update-user`

Header:

```text
Authorization: Bearer <Token>
```

Request:

  "ITEM": "ADD_USER",
  "USER_SYS_ID": 0,
  "SYSTEM_ROLE_SYS_ID": 1,

## Handoff KT for Backend Developer

Walkthrough order:

1. Start at `index.js`.
2. Show only active City Survey routes are mounted.
3. Show `config/db.js` reads all DB config from `.env`.
4. Show `routes/auth.js`:
   - normalizes auth payload
   - calls auth procedure
   - parses `JSON_VALUE`
   - signs JWT
5. Show `middlewares/auth.js`:
   - expects `Authorization: Bearer <token>`
   - returns 401 for missing token
   - returns 403 for invalid token
6. Show `routes/user.js`:
   - requires JWT middleware
   - calls user list procedure for view/list
   - validates `ITEM`
   - wraps payload into array for DB
   - calls user profile procedure
7. Show `routes/customer.js`:
   - requires JWT middleware
   - calls customer list procedure for view/list
   - validates `CUSTOMER_SYS_ID` and calls specific customer procedure
   - validates `ITEM` and required `ADD` fields
   - calls customer details procedure
8. Show `routes/project.js`:
   - requires JWT middleware
   - calls project list procedure for view/list
   - validates `ITEM` and required `ADD` fields
   - calls project procedure
9. Show `routes/master.js`:
   - requires JWT middleware
   - calls master list procedure for system roles
   - validates `ITEM` and required `ADD` fields
   - calls master add/update procedure for system roles
10. Show `routes/health.js`:
   - returns API process status without DB or JWT
11. Show `utils/procedureResponse.js`:
   - parses procedure response safely
12. Show Swagger at `/api/api-docs`.
13. Run health, auth, user, customer list, project list, system role list, safe
    customer validation, safe project validation, and safe system-role
    validation demo.
